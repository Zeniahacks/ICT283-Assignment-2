DESIGN RATIONALE:

BST USAGE:
Primary data structure for storing all weather records. Provides natural chronological ordering by date with efficient O(log n) search, insert, and range queries. Function pointers enable flexible traversal strategies without modifying core BST logic. Template design allows reuse in other contexts.

PROS:
- Maintains sorted order automatically
- Efficient for date-based queries
- Flexible traversal with function pointers
- Clean separation of data and operations

CONS:
- Can become unbalanced with sorted data
- More complex than simple arrays
- Additional memory overhead for pointers

MAP USAGE:
Custom Map class demonstrates understanding of essential map operations. Used for bonus eligibility demonstration with minimal interface focusing on core functionality.

PROS:
- Clean, focused abstraction
- Template-based for type safety
- Minimal interface demonstrates design understanding
- Eligible for bonus marks

CONS:
- Limited functionality compared to full std::map
- Additional layer of abstraction

STATISTICAL DECOUPLING:
